# HRMS 档案页 Glass Aurora 改造设计

## 背景

当前 `我的档案` 页面虽然已经引入了 aurora blob 和玻璃卡片，但真实观感仍偏黑、偏实。核心原因不是单一底板颜色，而是三层叠加：

1. `#profile-page` 自己的 aurora 背景透明度偏弱，光斑尺寸与扩散半径不足。
2. 页面后续又复用了同一文件里偏“黑金金融面板”的统一背景体系，造成更重的暗底和额外遮压。
3. 档案页核心卡片的底色与阴影都偏实，玻璃层透感不足，无法让背景极光穿透到前景。

本次目标不是“稍微提亮”，而是尽量贴近 `Glass Aurora · Brand Book` 的目标状态：让用户先感知到空间里的光，再感知到玻璃卡片和信息层级。

## 目标

本次仅先改造 `我的档案` 页面，目标如下：

1. 背景恢复为品牌书定义的三色极光底：左上 Iris、右上 Coral、底部 Mint。
2. 档案页脱离现有“金融黑金面板”视觉体系，不再继承压暗的通用背景。
3. 头部信息卡、通知区、档案信息卡、考勤概览卡和快捷操作卡统一改成更通透的 glass card。
4. 功能色只承担语义强调，不再用大面积厚重色块压住 aurora。
5. 在手机端优先保证光感、可读性和层级清晰，不牺牲当前主要内容结构。

## 设计方案

### 1. 页面背景

`#profile-page` 使用独立背景，不再沿用后面那组 `--rep-bg0 / --rep-bg1 / --rep-gold` 的金融面板背景。背景由以下三层组成：

1. 左上大半径 Iris 光斑，透明度明显提高。
2. 右上 Coral 光斑，位置更靠近人物和通知区，形成明显偏暖光感。
3. 底部 Mint 光斑，承担冷色空气感与平衡作用。

底层保留深空色系，但只作为承载，不再成为主观感知主体。颗粒噪点保留，作为轻微表面质感。

### 2. 玻璃卡片

所有档案页主卡片统一使用同一套 glass recipe：

1. 更浅的半透明底色，而不是深灰实体底。
2. 更强的 `backdrop-filter: blur(...) saturate(...)`，让卡片真正“吃到”背后的 aurora。
3. 更明显的高光边、轻微内反射和柔和外阴影，形成玻璃层。
4. 卡片之间保持同一圆角、边框与高光逻辑，避免局部风格跑偏。

### 3. 页面重点区域

#### 头部人物卡

头部卡要成为整页最能体现 Glass Aurora 的区域：

1. 人物头像底从普通蓝紫渐变升级为更贴近 Iris/Coral 的透光渐变。
2. 门店级别 badge 继续保留，但也切到玻璃语义，不用灰实底。
3. 文本层级保持清楚，避免极光直接压到文字后面。

#### 通知区

通知区保持现有结构，但通知列表卡片切换为浅透玻璃片层，让这一段不再像深色内容盒。

#### 档案信息卡 / 考勤概览卡

这两块是当前“发黑感”的主要来源。调整方向：

1. 内部指标小卡也做轻量玻璃化，不再是深灰小砖块。
2. 语义数值用 Iris / Mint / Coral / Amber 做高亮。
3. 保留现有信息密度，但通过背景、边框和明度关系拉开层级。

#### 快捷操作卡

快捷操作卡保留现有布局与文案，只改视觉体系：

1. 提高玻璃感与 hover 态边框亮度。
2. 图标底板切换为更轻的功能色发光片。
3. 离职申请维持弱化，但弱化方式从“整体发灰发闷”改成“仍在玻璃体系内的低强调”。

## 实施边界

本次只动 `我的档案` 页面相关样式与必要结构，不改：

1. 档案页现有数据接口。
2. 页面内功能逻辑与跳转行为。
3. 其他模块的黑金视觉体系。

如果后续用户认可这版效果，再考虑将同一 aurora 体系扩展到员工管理、考勤或其他页面。

## 验证标准

完成后通过以下标准判断是否达标：

1. 第一眼感知应是“极光 + 玻璃”，而不是“黑底页面”。
2. 三色 aurora 在不影响阅读的情况下可明确感知。
3. 主要卡片能明显看到透感和高光，不再像厚重实体面板。
4. 手机端档案页仍然清晰、稳定，不出现文字对比度不足或信息断层。
